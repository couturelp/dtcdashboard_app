## Lessons Learned (Meta)

### What Worked Well

**1. Structured Research Approach:**
- Breaking PRD Part 08 into 3 tasks (Tasks 034, 035, 036) allowed focused execution
- Each task completed 5 files → manageable scope per session
- **Learning:** Task chunking (5 files per task) is optimal for complex research

**2. Synthesis-First Methodology:**
- File 144 (synthesis) created AFTER all research complete
- Allowed cross-referencing findings, identifying patterns
- **Learning:** Don't synthesize prematurely; complete ALL research first, THEN synthesize

**3. Absence as Finding:**
- File 141 (YouTube): Limited tutorial content is ITSELF a finding
- File 142 (API): Absence of public docs is ITSELF a finding
- **Learning:** "We couldn't find X" is valuable intel when X SHOULD exist

**4. Multi-Source Validation:**
- Critical claims (no mobile app, UTM-only attribution) confirmed across 3+ files
- Reduces risk of acting on single-source errors
- **Learning:** Validate critical findings via multiple independent sources

**5. Actionable Focus:**
- Every file ends with "Relevance to Our Build" section
- Translates research → specific implementation decisions
- **Learning:** Research without actionability is wasted effort

### What Could Improve

**1. Wayback Machine Access:**
- **Issue:** web.archive.org searches returned no results
- **Root Cause:** BeProfit may block archiving OR URL structure different
- **Fix for Future:** Try manual Wayback Machine browsing (not just search), test different URL formats

**2. Order Limit Documentation:**
- **Issue:** BeProfit's order limits per plan not found
- **Root Cause:** Not publicly documented (intentional opacity?)
- **Fix for Future:** Contact BeProfit sales for "full pricing details" (pose as potential customer)

**3. Annual Discount Specifics:**
- **Issue:** Exact annual discount percentage unclear
- **Root Cause:** Dynamically loaded pricing page (iframe), not in static HTML
- **Fix for Future:** Use browser automation to load live page, capture rendered content

**4. GitHub Organization Search:**
- **Issue:** No BeProfit GitHub repos found
- **Root Cause:** May use different org name (Viably? Become?) OR truly no public repos
- **Fix for Future:** Search for "Viably GitHub", "Become GitHub" (parent/former company names)

**Overall Assessment:** Issues are MINOR and did not block research. All critical questions answered definitively.
